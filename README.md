# 3_办案体系（蓝色系统）

本体系是整个“五色一体化追货款系统”的核心，是从“起诉前评估 → 诉讼 → 判决 → 执行”全链路的办案作业系统。

它一方面是前端流量与成交的“承载底座”，  
另一方面又是后端口碑、转介绍、复购与裂变的“发射平台”。

---

## 1. 办案体系在整体业务中的定位

- 漏斗底座：  
  所有前端通过流量、邀约、成交带来的案源，最终都要落到“办案”环节来完成交付和变现。

- 口碑金字塔的起点：  
  办案质量 → 回款时间与效率 → 客户满意度 → 转介绍、复购、口碑传播。

- 主任的抓手：  
  对于一个“所中所”模式的团队，主任不可能事事亲自办案，  
  但必须有一套可以“远程控质量”的抓手。  
  这套抓手以 **三大质控节点 + 三张评估表** 为核心。

---

## 2. 办案体系七大阶段（诉讼主线）

> 说明：这是从“起诉前”到“执行完毕”的一整条主线 SOP，所有案件默认遵循这一条主线。

1. 阶段1：接受代理、案件评估与诉讼策略  
2. 阶段2：证据收集、材料整理、立案与保全准备  
3. 阶段3：立案受理、保全结果、是否追加保全  
4. 阶段4：诉前强制调解（第一道和解防线）  
5. 阶段5：庭前准备、答辩与第二道和解防线  
6. 阶段6：开庭、合议、判决、判后和解（第三道和解防线）  
7. 阶段7：执行立案、执行和解、强制执行、终本/中本处理  

> 在此基础上，另设一个“执行专项线”作为补充：  
> - 只接执行案件（终本/中本再执行）的专项 SOP（见本体系下的《执行专项》部分）。

---

## 3. 三大质控节点与三道和解防线

为解决“案件多、分工细、主任如何控质量”的问题，本体系设计了 **三大质控节点 + 三张标准化评估表**，与三道和解防线相互呼应：

### 3.1 质控节点一：立案前案件评估表（起诉前评估）

- 对应阶段：  
  - 阶段1：接受代理、初步评估  
  - 阶段2：证据整理、诉讼方案成型前

- 主要作用：  
  - 判断案件是否值得打、值不值得保全、是否适合风险代理或中止谈判；  
  - 梳理案情要点与风险点，为第一道和解防线（诉前调解）提供底层判断。

- 输出表单：  
  - 《诉前案件评估与诉讼策略审核表》（主任审核用）  
  - 用于：主任把关“案件方向、策略及胜算”。

### 3.2 质控节点二：庭前答辩与开庭准备评审表

- 对应阶段：  
  - 阶段5：庭前准备、证据补强、答辩状成稿  
  - 紧接着即将开庭

- 主要作用：  
  - 对已经成型的诉讼思路、证据体系、答辩逻辑进行二次评估；  
  - 对 “第二道和解防线” 的条件（是否适合进一步谈和解）做出判断。

- 输出表单：  
  - 《庭前答辩与开庭准备评审表》  
  - 用于：主任或资深律师对“开庭质量”与“和解时机”进行把关。

### 3.3 质控节点三：判决后执行评估表（执行前评估）

- 对应阶段：  
  - 阶段7：判决生效后，进入执行程序之前  
  - 包含：是否值得申请强制执行、是否适合执行和解、是否适合终本或暂缓等。

- 主要作用：  
  - 在执行前，对“现实可回款性”“成本与收益”进行评估；  
  - 是“第三道和解防线”与“是否单独走执行专项线”的分水岭。

- 输出表单：  
  - 《判决后执行可行性评估与执行方案审核表》  
  - 用于：主任对执行阶段的投入产出、策略组合进行把关。

> 这三张表，是本办案体系的三道“质量闸门”，  
> 也是主任控制案件质量、回款效率与整体口碑的三大抓手。

## 4. 目录结构与子模块说明（蓝色系统）

本目录下规划的二级结构如下：

- `3.1_总则与办案理念/`  
- `3.2_时间节点与节奏总纲/`  
- `3.3_阶段1_接受代理_评估_诉讼策略/`  
- `3.4_阶段2_证据收集_立案_保全/`  
- `3.5_阶段3_立案受理_保全结果_追加保全/`  
- `3.6_阶段4_诉前调解_第一道和解防线/`  
- `3.7_阶段5_庭前准备_第二道和解防线/`  
- `3.8_阶段6_开庭_判决_判后和解_第三道防线/`  
- `3.9_阶段7_执行立案_执行和解_强制执行/`  
- `执行专项_终本_中本再执行SOP/`  
- `案件管理_质检_复盘系统/`

> 后续在各阶段目录下，将补充：  
> - 具体 SOP 文档  
> - 表格表单（含三大评估表）  
> - 节奏控制清单（H 小时 / Day 天）  
> - 质检 checklist。

---

## 5. 使用方式（对团队成员）

1. **办案律师 / 助理**  
   - 按阶段查看对应目录下的 SOP 与模板；  
   - 在关键节点前，先完成对应评估表，再交主任 / 资深律师审核。

2. **主任 / 负责人**  
   - 主抓“三大质控节点”的评估表：诉前评估、庭前评审、执行前评估；  
   - 以此为抓手控制案件质量、回款周期、客户满意度。

3. **案管 / 运营**  
   - 负责时间节点（H / Day）在系统中落地：录入、提醒、督办；  
   - 将节点数据用于事后复盘与绩效考核。
