# 3.2 时间节点与节奏总纲（H 小时 + Day 天）

本模块用于统一本所办案的“时间观念”，  
区分三个层次的时间控制：

1. 我方可控的“小时级（H）节点”  
2. 我方可控的“天级（Day）节点”  
3. 法院不可控，但我方可控“跟进频率”的节点  

并与“三大质控评估表”相配合，形成一个真正可执行、可追责的办案节奏系统。

---

## 1. H 小时级节点（接案–签约–证据初筛）

> 起点定义：T0 = 签订代理合同 / 确认委托关系之时。

### 1.1 当天必须完成的动作（T0 当日内）

- 将案件基础信息录入双系统（CRM + 案管系统）。  
- 完成客户基本信息与欠款要素录入（金额、对方主体、合同关系）。  
- 完成一次“粗筛式证据整理”：  
  - 有无书面合同 / 订单  
  - 有无对账单、发票、送货单  
  - 有无聊天记录 / 邮件往来  
- 与客户确认收费模式与基本预期：  
  - 是否风险代理  
  - 是否需要保全  
  - 大致回款时间预期沟通。

### 1.2 24 小时内节点（T0+24H）

- 办案律师完成第一次“案件体检”：  
  - 案件基本事实是否清晰  
  - 是否存在重大程序性风险 / 实体性风险  
  - 初步判断：适合诉讼、适合和解还是暂缓。

- 形成《诉前案件评估与诉讼策略审核表（初稿）》：  
  - 由办案律师填写  
  - 提交主任进行第一次评估与批示

> 说明：  
> 此节点主要是“案件方向与风险的快速把关”，  
> 对应 **三大质控节点中的第一道闸门（诉前评估表）** 的初版。

---

## 2. Day 天级节点（我方可控的流程）

在 H 小时基础上，进入“以天为单位”的节奏控制。

核心若干标准节点建议如下（可按实际再细化）：

### 2.1 T0+3 日内：诉讼策略定稿与第一次和解评估

- 办案律师完成：  
  - 案件详细事实整理  
  - 证据清单初稿  
  - 管辖确定  
  - 保全必要性初步判断  

- 主任基于《诉前案件评估与诉讼策略审核表》进行审批：  
  - 批示：  
    - 是否立案  
    - 是否同步推进和解尝试  
    - 是否同步申请保全  

### 2.2 T0+10 日内：起诉 + 保全材料提交完成

- 起诉状定稿  
- 证据目录与复印件整理完毕  
- 保全申请书及财产线索准备完成（如需）  
- 立案窗口 / 网上立案提交完毕  
- 保全费（如需）缴纳完毕  

这是“立案前可控时间”的最后一道质量防线，  
逾期必须在系统中标记原因，并由办案律师作书面说明。

## 3. 法院不可控节点 + 我方可控跟进节奏

法院的立案、排期、审理与裁判时间整体不可控，  
但我方可以对“跟进频率与汇报节奏”进行标准化。

### 3.1 立案后

- 立案当天：  
  - 在案管系统中录入立案时间与案号。  
- 每 7–10 日：  
  - 查看系统进度 / 联系立案庭确认状态。  
- 每 15–20 日：  
  - 如长期未进入下一环节，向法院发起一次温和催办。

### 3.2 排期与开庭前

- 收到排期通知当日：  
  - 录入开庭时间、地点、承办法官。  

- 距离开庭 7–10 日：  
  - 完成《庭前答辩与开庭准备评审表》  
  - 主任或资深律师进行质控：  
    - 证据是否完整  
    - 法律关系是否梳理清楚  
    - 争议焦点是否提炼到位  
    - 是否具备再次谈和解的最佳时机  

此处对应 **第二大质控节点**。

### 3.3 判决后至执行前

- 判决书送达当日：  
  - 录入判决生效起算日期。  

- 判决生效后 7 日内：  
  - 完成《判决后执行可行性评估与执行方案审核表》。  
  - 内容包括：  
    - 被执行人财产状况现状  
    - 可回款性评估  
    - 执行措施组合建议  
    - 是否建议执行和解 / 立即执行 / 暂不执行 / 放弃。

此处对应 **第三大质控节点**，  
是“是否进入执行、是否走执行专项线”的关键闸门。

---

## 4. 三大质控节点与时间节奏的结合

可理解为三个分界点：

1. **诉前节点（T0+3 日左右）**  
   - 对应：《诉前案件评估与诉讼策略审核表》  
   - 目标：案件方向、诉讼/和解/保全策略清晰。

2. **庭前节点（开庭前 7–10 日）**  
   - 对应：《庭前答辩与开庭准备评审表》  
   - 目标：证据与法理准备充分；判断是否再次对被告发起和解攻势。

3. **执行前节点（判决生效后 7 日内）**  
   - 对应：《判决后执行可行性评估与执行方案审核表》  
   - 目标：判断执行投入产出比，决定执行路径与是否走执行专项。

这三张表，将来会各自独立成文档与表单模板，  
在本模块中只做总则说明。
